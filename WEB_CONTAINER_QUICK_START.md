# WebContainer 快速开始指南

## 🎯 什么是 WebContainer？

WebContainer 是 PageForge 的核心功能之一，它允许您在浏览器中直接运行和测试生成的 HTML 页面。与传统的 iframe 预览不同，WebContainer 提供了完整的 Node.js 开发环境，包括：

- ✅ **完整的 npm 包管理**
- ✅ **真实的开发服务器**
- ✅ **交互式 JavaScript 执行**
- ✅ **控制台输出和调试**

## 🚀 快速体验

### 步骤 1: 启动应用

```bash
# 启动后端
cd backend && uvicorn app.main:app --reload

# 启动前端
cd frontend && npm run dev
```

### 步骤 2: 创建页面

1. 打开 http://localhost:6000
2. 在聊天面板中输入："创建一个计数器应用"
3. 等待 AI 生成 HTML 代码

### 步骤 3: 启动 WebContainer

1. 在右侧预览面板中，点击 "运行环境" 标签
2. 点击 "启动环境" 按钮
3. 等待系统自动完成：
   - 🔄 初始化 WebContainer
   - 📁 创建项目文件
   - 📦 安装 npm 依赖
   - 🚀 启动开发服务器

### 步骤 4: 交互测试

- 🎉 在完整的浏览器环境中测试页面
- 📱 切换不同设备尺寸（桌面/平板/手机）
- 🔧 查看控制台输出和调试信息

## 💡 使用技巧

### 选择合适的预览模式

| 模式 | 适用场景 | 特点 |
|------|----------|------|
| **预览** | 快速查看效果 | 即时加载，轻量级 |
| **运行环境** | 测试交互功能 | 完整环境，支持 JS |
| **源码** | 查看和编辑代码 | 语法高亮，易于阅读 |

### 性能优化

- ⚡ **首次启动较慢**：WebContainer 初始化需要 30-60 秒
- 🔄 **后续启动更快**：浏览器会缓存部分资源
- 💾 **及时清理**：使用完毕后点击 "停止" 释放内存

### 故障排除

#### 启动失败

```
❌ 错误：WebContainer 初始化失败
✅ 解决方案：
   1. 检查网络连接
   2. 刷新页面重试
   3. 查看浏览器控制台错误
```

#### 依赖安装失败

```
❌ 错误：npm install 失败
✅ 解决方案：
   1. 检查 package.json 配置
   2. 简化项目依赖
   3. 重启 WebContainer
```

## 🔧 高级功能

### 控制台监控

1. 点击 "控制台" 按钮
2. 实时查看应用输出
3. 监控错误和警告信息
4. 调试 JavaScript 代码

### 环境控制

- **重启**：重新加载整个环境
- **停止**：完全关闭 WebContainer
- **状态查看**：实时显示运行状态

### 多尺寸测试

- **桌面端**：100% 宽度，适合开发调试
- **平板端**：768px 宽度，测试中等屏幕
- **手机端**：375px 宽度，测试移动设备

## 📚 最佳实践

### 开发流程

1. **快速原型** → 使用 "预览" 模式快速查看
2. **功能测试** → 切换到 "运行环境" 测试交互
3. **调试优化** → 使用控制台查看错误和性能
4. **最终确认** → 多尺寸测试确保响应式效果

### 性能建议

- 🎯 **按需启动**：只在需要测试交互时启动 WebContainer
- 🗑️ **及时清理**：测试完成后及时停止释放资源
- 📦 **简化依赖**：避免不必要的 npm 包减少启动时间

### 调试技巧

- 🔍 **使用控制台**：查看详细的运行日志
- 📱 **多设备测试**：确保在不同尺寸下都能正常工作
- ⚡ **性能监控**：观察启动时间和内存使用
