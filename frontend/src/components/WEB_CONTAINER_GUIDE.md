# WebContainer 功能使用说明

## 概述

WebContainer 是一个强大的浏览器内开发环境，允许您在浏览器中直接运行和测试生成的项目。与传统的 iframe 预览不同，WebContainer 提供了完整的 Node.js 环境，支持 npm 包安装、构建工具和实时开发服务器。

## 功能特点

### 🚀 完整的开发环境
- **Node.js 运行时**: 完整的 Node.js 环境，支持 npm 包管理
- **文件系统**: 虚拟文件系统，支持文件读写操作
- **进程管理**: 支持启动和管理多个进程
- **网络访问**: 支持端口映射和外部访问

### 💻 开发工具支持
- **包管理器**: 完整支持 npm 依赖安装
- **构建工具**: 支持 Vite、Webpack 等现代构建工具
- **开发服务器**: 自动启动开发服务器，支持热重载
- **控制台输出**: 实时查看应用日志和错误信息

### 🎯 交互功能
- **启动/停止控制**: 可以随时启动或停止 WebContainer
- **实时预览**: 代码更改立即反映在预览中
- **响应式测试**: 支持桌面、平板、手机多尺寸预览
- **控制台监控**: 实时查看应用运行状态和输出

## 使用方法

### 1. 基本使用

1. **生成 HTML 内容**: 通过 AI 生成页面内容
2. **切换到运行环境**: 在预览面板中点击 "运行环境" 标签
3. **启动容器**: 点击 "启动环境" 按钮
4. **等待初始化**: 系统会自动完成以下步骤：
   - 初始化 WebContainer
   - 创建项目文件结构
   - 安装依赖包
   - 启动开发服务器
5. **交互测试**: 在运行环境中与生成的页面进行交互

### 2. 高级功能

#### 控制台监控
- 点击 "控制台" 按钮查看实时输出
- 监控应用启动状态和运行日志
- 查看错误信息和调试输出

#### 环境控制
- **重启**: 点击 "重启" 按钮重新加载环境
- **停止**: 点击 "停止" 按钮关闭运行环境
- **状态查看**: 实时显示容器运行状态

#### 多视图切换
- **预览模式**: 传统 iframe 预览，快速查看效果
- **运行环境**: WebContainer 完整环境，支持交互
- **源码查看**: 查看生成的 HTML 源代码

## 技术实现

### 文件结构转换

当从 HTML 创建 WebContainer 项目时，系统会自动：

1. **解析 HTML**: 分离结构、样式和脚本
2. **创建文件结构**:
   ```
   /index.html          # 主页面文件
   /src/style.css       # 提取的样式
   /src/main.js         # 提取的脚本
   /package.json        # 项目配置
   ```

3. **依赖管理**: 自动添加常用依赖
   ```json
n   {
     "dependencies": {
       "react": "^19.0.0",
       "react-dom": "^19.0.0"
     },
     "devDependencies": {
       "vite": "^5.0.0"
     }
   }
   ```

### 生命周期管理

1. **初始化阶段**:
   - 创建 WebContainer 实例
   - 设置虚拟文件系统
   - 准备运行环境

2. **项目创建**:
   - 写入 HTML、CSS、JS 文件
   - 生成 package.json
   - 创建必要的目录结构

3. **依赖安装**:
   - 执行 `npm install`
   - 下载和安装所有依赖
   - 准备构建环境

4. **服务器启动**:
   - 执行 `npm run dev`
   - 启动开发服务器
   - 建立端口映射

5. **资源清理**:
   - 停止所有运行进程
   - 清理临时文件
   - 释放内存资源

## 注意事项

### 性能考虑
- WebContainer 启动需要一定时间（通常 30-60 秒）
- 大项目可能需要更多内存和计算资源
- 建议在必要时才启动运行环境

### 限制说明
- 某些 Node.js API 可能受限
- 文件系统是虚拟的，重启后数据会丢失
- 网络访问可能有跨域限制

### 故障排除

1. **启动失败**:
   - 检查网络连接
   - 查看控制台错误信息
   - 尝试重启容器

2. **依赖安装失败**:
   - 检查 package.json 配置
   - 确认依赖包名称正确
   - 尝试更新依赖版本

3. **服务器无法访问**:
   - 确认端口配置正确
   - 检查防火墙设置
   - 查看服务器日志

## 最佳实践

1. **合理使用**: 仅在需要完整环境测试时才启动 WebContainer
2. **及时清理**: 使用完毕后及时停止容器释放资源
3. **错误监控**: 密切关注控制台输出，及时发现和解决问题
4. **版本控制**: 重要项目建议在本地保存代码

通过 WebContainer，您可以在浏览器中获得接近本地开发的完整体验，大大提升了页面生成和测试的效率。
